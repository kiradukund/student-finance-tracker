# Wireframe Notes

## Files
- `mobile.jpg` — layout at 360px width
- `tablet.jpg` — layout at 768px width
- `desktop.jpg` — layout at 1024px width

## Section Order (all screen sizes)
1. Skip link
2. Header — app name
3. Nav — links to all sections
4. About — purpose, name, email
5. Dashboard — stats, 7-day chart, aria-live budget status
6. Records — search, sort, table (cards on mobile)
7. Add/Edit Form — all input fields
8. Settings — currency rates, spending cap
9. Footer

## Layout Changes Across Breakpoints

**Mobile (360px)**
- Nav links stack vertically
- Records shown as stacked cards
- Form fields full width
- Dashboard stats in single column

**Tablet (768px)**
- Nav links horizontal
- Dashboard stats in 2-column grid
- Records shown as a table
- Form fields full width

**Desktop (1024px)**
- Content centred with max-width
- Dashboard stats in single row of 4
- Table shows all columns
- 7-day bar chart visible in dashboard

## Accessibility Notes on Wireframes
- Skip link drawn at top of every sketch
- Every form input drawn with a visible label
- aria-live region labelled in Dashboard section
- Edit and Delete are buttons in every layout
